# 第 6 课作业：Prompt 与状态，决定 Agent 怎么想、记什么

## 作业目标

区分 Prompt 与 State 的职责边界，并理解为什么当前项目中的状态模型还处在逐步接入阶段。

## 练习 1：让 `executionPlan` 真的被看见

请你思考，如果想把 `executionPlan` 展示到 UI 中，至少要完成哪几个步骤：

- 谁来写入计划
- 写到哪里
- UI 从哪里取
- 以什么形式展示

这个练习的重点不是立刻实现，而是训练你把“状态设计”变成“可见系统行为”。

### 提交要求

- 至少拆成 4 个步骤
- 指出状态写入层和展示层分别在哪里
- 说明这件事为什么现在还没自动成立

### 完成标准

你能够把一个抽象状态字段转化成清晰的工程落地方案。

## 练习 2：拆解 Prompt 内容

请你把当前 Prompt 中的规则尝试分成两类：

- 通用型规则
- 项目型规则

思考它们未来是否适合拆成模板、片段或不同层级的 Prompt 组合。

### 提交要求

- 两类规则都至少列出 3 条
- 说明分类依据
- 说明为什么这种拆法有助于后续演进

### 完成标准

你能够从 Prompt 工程角度重新组织当前规则，而不是只把它看成一整段文本。

## 自查问题

在提交前，确认自己能回答：

- Prompt 和 State 的本质区别是什么？
- 为什么 Prompt 只能提供软约束？
- 当前 `AgentState` 为什么说还不是唯一状态中枢？

# 第 5 课作业：把 LLM、工具和会话真正组装成 Agent

## 作业目标

理解 `src/agent/agent.ts` 如何把模型、工具、Prompt、会话与运行包装层整合为一个真正可执行的 Agent。

## 练习 1：解释 `createAgent()` 的返回值

请你尝试用自己的语言说明下面几个值分别服务于谁：

- `agent`
- `sessionId`
- `workingDirectory`
- `fileCache`
- `tools`

要求不是翻译变量名，而是说清楚它们在系统中的角色。

### 提交要求

- 每个值至少写 1 句职责说明
- 至少指出 1 个“谁在使用它”
- 不要只做字面翻译

### 完成标准

你能够从系统集成视角解释 Agent 装配结果，而不是只记住返回字段名。

## 练习 2：增加一个运行参数

设想你要增加一个新的运行参数，比如是否开启某类工具调试输出。请思考：

- 它应该从 CLI 哪一层传入
- 如何进入最终配置
- 是否要影响 `createAgent()` 的装配过程

### 提交要求

- 画出或写出参数流转路径
- 至少指出会影响的 2 个模块
- 说明为什么放在那个层级最合理

### 完成标准

你能够说明一个新的运行配置是如何从入口进入 Agent 装配过程的。

- `createSystemPrompt()` 为什么要在工具准备好之后再调用？

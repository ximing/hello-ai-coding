# 第 4 课作业：工具系统，让 Agent 真正能动手

## 作业目标

理解工具系统如何把大模型升级为可执行的 Agent，并掌握设计一个工具时最基本的检查框架。

## 练习 1：设计一个 grep 风格工具

请你尝试设计一个新工具，不要求立刻实现，只需要定义下面几项：

- 工具名
- 工具描述
- 输入参数 schema
- 输出格式
- 安全边界

目标是让它比当前 `search_code` 更适合精确查找。

### 提交要求

- 用结构化方式写出工具定义
- 说明它和现有 `search_code` 的差异
- 至少说明 1 个它更适合解决的场景

### 完成标准

你能够从“能力协议”视角去设计工具，而不只是写一个函数名字。

## 练习 2：比较 `read_file` 与 `search_code`

请你回答：

- 什么情况下优先用 `read_file`
- 什么情况下优先用 `search_code`
- 为什么二者不能相互完全替代

### 提交要求

- 至少各举 1 个使用场景
- 说明二者的输入、输出和适用粒度差别
- 不要只说“一个读文件，一个搜代码”

### 完成标准

你能够理解不同工具的职责边界，而不是把所有工具都看成“反正都能帮模型做事”。

## 自查问题

在提交前，确认自己能回答：

- 一个 Agent 工具最少要包含哪四部分？
- 为什么 schema 和 description 都很重要？
- 为什么 createAgent 要先准备工具，再创建 Agent？

# 第 7 课作业：流式 UI，把 Agent 的过程展示出来

## 作业目标

理解 UI 如何把 Agent 的执行过程可视化，并学会从“可观察性”角度评价一个 Agent 界面。

## 练习 1：增强工具结果展示

请你尝试设计一种更适合教学场景的工具结果展示方式。你可以思考：

- 是否应该显示完整结果
- 是否需要折叠展开
- 是否区分 stdout 和 stderr
- 是否需要按工具分组

### 提交要求

- 画出或文字描述展示结构
- 至少指出 2 个当前展示方式的不足
- 说明你的方案更适合教学的原因

### 完成标准

你能够从 UI 信息结构角度改进当前工具结果呈现，而不是只想着“把字显示出来”。

## 练习 2：增加处理中指示器

请你设想如何在 UI 中加入更明显的处理中状态，例如：

- 旋转指示器
- 步骤计数
- 正在执行第几个工具
- 当前阶段标签

目标是让用户一眼知道系统不是卡住了，而是在工作。

### 提交要求

- 说明要新增哪些状态字段或 UI 逻辑
- 指出这些信息应来自哪里
- 说明最终展示在界面的哪个区域

### 完成标准

你能够把“处理中提示”设计成一个真正可实现的 UI 方案，而不是停留在概念描述。

## 自查问题

在提交前，确认自己能回答：

- 为什么 Agent 特别需要过程型 UI？
- `handleSubmit()` 为什么是 UI 和 Agent 链路的交汇点？
- 为什么最终答案和中间过程应该分开展示？

# 第 3 课作业：配置系统与安全边界

## 作业目标

理解 Agent 的配置合并策略，以及命令白名单和路径沙箱为何是最基础的安全边界。

## 练习 1：扩展命令白名单

请你尝试在 `allowedCommands` 中增加一个你认为相对安全的命令，并思考：

- 它真的安全么？
- 它是否会间接带来新风险？
- 如果允许这个命令，是否还需要参数级限制？

### 提交要求

- 写出你要添加的命令
- 说明选择它的理由
- 说明新增后可能带来的风险变化

### 完成标准

你能够意识到“进入白名单”不等于“完全安全”，并开始从能力与风险同时评估 Agent 工具边界。

## 练习 2：解释路径沙箱

请你尝试向一个完全不了解 Agent 的同学解释：为什么 `validatePath()` 一定要限制工作目录之外的路径。

要求不要只说“为了安全”，而要说出具体风险场景。

### 提交要求

- 至少举 2 个具体风险场景
- 要说明“为什么工作目录是合理边界”
- 不能只写抽象结论

### 完成标准

你能够把路径沙箱讲成一个真实系统边界，而不是一句口号。

## 自查问题

在提交前，确认自己能回答：

- 当前配置有哪几层来源？
- 为什么配置要采用覆盖式合并？
- 当前项目已经具备哪些安全能力？
- `requireConfirmation` 为什么说还是一个未闭环能力？

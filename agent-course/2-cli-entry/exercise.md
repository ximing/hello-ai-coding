# 第 2 课作业：从 CLI 入口搭起 Agent 外壳

## 作业目标

理解命令行入口如何接收用户输入、整理配置，并把运行控制权交给 Agent 与 Ink UI。

## 练习 1：新增一个调试命令

目标：增加一个类似 `show-config` 的子命令，只打印当前运行配置，不进入 Agent 执行。

### 思考要求

在实现前先回答下面几个问题：

- 这个命令应该复用 `loadConfig()` 吗？
- 它要不要支持 `--config` 和 `--working-dir`？
- 它和 `dry-run` 的区别是什么？

### 进阶要求

如果你愿意继续做，可以真的在 `src/cli.ts` 里设计这个命令，并说明它放在 CLI 哪一层最合理。

### 完成标准

你能够清楚说明一个新 CLI 命令应如何接入当前项目，而不是只停留在口头设想。

## 练习 2：解释 dry-run 的价值

请你用自己的语言解释，为什么很多工程工具都需要“预览模式”。

### 提示方向

- 避免误操作
- 帮助调试参数
- 提升用户信心

### 提交要求

- 使用 100 到 200 字说明
- 至少举 1 个与你熟悉的工程工具相关的例子
- 明确说明 dry-run 与真正执行的边界

### 完成标准

你能够解释为什么入口层不只负责启动，还负责控制是否真的进入执行链路。

## 自查问题

在提交前，确认自己能回答：

- `src/cli.ts` 的核心职责是什么？
- 命令行参数为什么要先归一化成配置覆盖项？
- `render(CliApp)` 为什么是交互式运行的起点？
